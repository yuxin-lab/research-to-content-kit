# Contributing Guide

Thank you for your interest in contributing to Research to Content Kit.

This project is an open-source toolkit for transforming research materials, expert interviews, and medical education content into structured public-facing science communication workflows.

## Project purpose

The project aims to provide practical templates, examples, checklists, and lightweight scripts for:

- Research paper summaries
- Short video scripts
- Slide outlines
- Expert interview structuring
- Social media post planning
- Bilingual Chinese-English science communication
- Compliance-aware wording review
- Public education content workflows

How to contribute a template

When adding a new template, please place it in:
templates/

A good template should include:

Basic information
Target audience
Communication goal
Suggested structure
Risk wording review
Safer wording examples
Final output section
Notes section
How to contribute an example

When adding a new example, please place it in:
examples/

A good example should include:

Topic
Source type
Target audience
Core message
Public-facing question
Structured output
Wording review
Related templates
How to contribute documentation

When adding documentation, please place it in:
docs/
Documentation may include:

Roadmaps
Prompt templates
Wording examples
Workflow explanations
Review notes
Platform-specific guidance
How to contribute scripts

When adding scripts, please place them in:

scripts/

Scripts should be lightweight, readable, and easy for non-technical users to understand.

Please include comments where helpful.

Medical and health communication notes

This project may include medical, health, skincare, nutrition, biotechnology, and science communication topics.

When contributing, please avoid wording that may imply:

Diagnosis
Treatment
Cure
Guaranteed effects
Replacement of medical care
Unsupported product claims
Overstated expert endorsement
Drug-like claims in product communication

Prefer safer wording such as:

May be related to
May help explain
Is associated with
From a science communication perspective
Individual situations vary
Professional evaluation may be needed
This content is for education only
Pull requests

Pull requests should be focused and easy to review.

Before submitting, please check:

The file is in the correct folder.
The wording is clear and responsible.
The content matches the project purpose.
Medical or health-related claims are not overstated.
Examples are educational and not misleading.
Issues

You can open an issue to suggest:

New templates
New examples
Wording improvements
Script ideas
Documentation improvements
Workflow improvements
Disclaimer

This project is for education, documentation, and science communication workflow purposes only.

It does not provide medical diagnosis, treatment advice, regulatory advice, or legal advice.
